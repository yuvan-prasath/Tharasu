Evolution Agent
Auto-generates harder questions for categories where all models struggle
🧬
How Evolution Works
1. Scans the database for tasks where all models scored below 5 on the judge scale
2. Groups failed tasks by category (math / coding / reasoning)
3. Calls llama3_70b as a question designer to generate 5 harder questions per category
4. Appends new questions to task_library.json — THARASU grows smarter over time
Run the Agent
This will query your DB and generate new benchmark questions.
Evolution Report
💡 Why "below 5"?
The judge scores on a 1–10 scale. Score ≥5 means the model handled the task adequately. Below 5 is a genuine failure. The agent only evolves categories where every model failed — signaling a universal blind spot in the task library, not just a single weak model.

The 7 Metrics
Every model response is graded across 7 dimensions
⏱
1. Latency
Time from API call to response, in milliseconds. Lower is better.
latency_ms = (time.time() - start) * 1000
🔢
2. Token Count
Total tokens consumed per response. Tracks efficiency across models.
response.usage.total_tokens
✅
3. Accuracy
For tasks with a known answer. String match with numeric fallback. Returns 100 or 0.
expected_answer in model_response
📊
4. BLEU Score
Text similarity between model output and reference answer. Range 0–1 via NLTK.
sentence_bleu([ref.split()], ans.split())
⚖️
5. LLM Judge Score
llama3_70b scores other models' answers 1–10. Self-evaluation skipped.
{{"score": 1–10, "reason": "..."}}
🔄
6. Consistency
Run each task 3× and compare judge scores. High variance = unreliable model.
1 - (std_dev / mean) of 3 runs
7. THARASU Score
Composite weighted formula — the signature metric of this framework
tharasu = (accuracy × 0.35)
         + (judge × 0.30)
         + (speed × 0.15)
         + (token_efficiency × 0.10)
         + (bleu × 0.10)
All inputs normalized to 0–100. Speed and token efficiency are inverted (lower = higher score).
